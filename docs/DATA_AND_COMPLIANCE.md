# Data and Compliance

## Data Sources
1. Metadata CSVs: `data/metadata/wikiart_csv/*`.
2. Images root: `data/raw/wikiart/...` (preferred) or `data/raw/images/...`.
3. Local archive option: `wikiart.zip` in repository root (auto-detected by pipeline).

## Manifest Semantics
Manifest columns:
1. `image_relpath`
2. `style_label`
3. `artist_label`
4. `genre_label`

Generated by joining task-specific split files on `image_relpath`.

## Download Utility
Use:
```bash
python scripts/download_images_from_manifest.py \
  --manifest data/manifests/train_quick.csv \
  --manifest data/manifests/val_quick.csv \
  --out-root data/raw/wikiart
```

## Rights and Redistribution
Read and enforce:
1. `DATA_RIGHTS.md`
2. `scripts/filter_redistributable.py`
3. `data/templates/provenance_template.csv`

Minimal policy:
1. Do not redistribute copyrighted full-resolution images without explicit rights.
2. Track provenance and rights status.
3. Filter release bundles to redistributable assets only.

## Provenance Workflow
1. Maintain a provenance CSV from `data/templates/provenance_template.csv`.
2. Split allowed vs blocked rows:
```bash
python scripts/filter_redistributable.py \
  --input data/templates/provenance_template.csv \
  --out-allowed outputs/provenance_allowed.csv \
  --out-blocked outputs/provenance_blocked.csv
```
